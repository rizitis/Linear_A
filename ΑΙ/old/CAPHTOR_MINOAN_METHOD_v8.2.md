## SECTION 1 — THE TRI-LAYERED MODEL

### 1.1 Core Thesis

Minoan Crete was NOT linguistically homogeneous. Linear A records three distinct sociolinguistic strata:

| Layer | Population | Biblical Source | Date | Contribution |
|-------|-----------|----------------|------|-------------|
| **Layer 1 — Substrate** | Japhethites / Pelasgians | Gen 10: Japheth → Javan | ~3000-2000 BCE | Vowels, divine names, acrophony, plant terminology |
| **Layer 2 — Superstate** | Hamites / Caphtorim | Gen 10: Ham → Mizraim → Caphtorim | ~2000-1700 BCE | Writing structure, CV-syllables, determinatives, bureaucracy |
| **Layer 3 — Influence** | Semitic traders | Phoenicians, Canaanites | ~1800-1400 BCE | Commercial vocabulary: KU-RO, M-L-K, Š-L-M |

### 1.2 The Substrate Law

Universal linguistic law: **religious vocabulary** (oldest layer) survives longest, while **commercial vocabulary** is replaced first. This explains why Linear A has:
- Aegean divine names (DA-MA-TE) — Layer 1, survived into Greek
- Semitic commercial roots (KU-RO) — Layer 3, replaced by Greek after ~1450 BCE

### 1.3 How to Apply the Model

When analyzing a Linear A sequence:

1. **First check Layer 1 (Aegean/Greek):** Does the sequence match a known pre-Greek substrate word or a Greek word with pre-Greek etymology? Look for -nthos, -ssos, -aro endings.
2. **Then check Layer 3 (Semitic):** Does the sequence, after applying R→L where valid, match a Semitic root (Hebrew, Akkadian, Phoenician)? Especially for commercial/administrative terms.
3. **Consider Layer 2 (Hamitic/Egyptian):** Is the sign a determinative or structural element borrowed from Egyptian writing practice?
4. **Best result:** When all three layers converge on the same reading.

### 1.4 What NOT to Assume

- Linear A is NOT a Semitic language — it has Semitic LOANWORDS
- Linear A is NOT Greek — it has pre-Greek SUBSTRATE words
- The writing SYSTEM was designed by Caphtorim (Layer 2) for an Aegean language (Layer 1) enriched with Semitic trade terms (Layer 3)

## SECTION 3 — TABLET READING PROTOCOL

### 3.0 Research Prerequisites [v8.1 — NEW]

**This section describes what to do BEFORE applying the decipherment algorithm. These are not analytical steps of the method — they are standard academic practice. They do not generate k\* scores or satisfy K-criteria. They provide CONTEXT, not CONCLUSIONS.**

Before applying Steps 0-8 to any tablet, consult existing scholarship:

| Source | What it provides | URL / Reference |
|--------|-----------------|-----------------|
| GORILA (Godart & Olivier) | Official transcription, facsimile, notes | Vols. 1-5 (print) |
| Younger's commentary | Phonetic transcription, scholarly notes per tablet | web archive / lineara.xyz |
| SigLA (Salgarella) | Palaeographic sign data, searchable | sigla.phis.me |
| Academic literature | Published interpretations, parallels | Scholar search / Academia.edu |

**Rules:**
- If prior interpretations exist, note them — but they enter the method as **hypotheses to test**, not as validated readings
- A published interpretation (e.g. Mosenkis 2015: KU-PI = "Cypriot") provides a **direction to investigate**, not a K-criteria score
- Only the K1-K7 criteria can validate a reading — scholarly consensus without K-criteria remains at the hypothesis level
- If you have web search capability, use it — but clearly separate what you FOUND from what the METHOD VALIDATES

**Rationale:** The ZA14/KU-PI incident (Feb 2026) demonstrated that an AI model following the Protocol mechanically but without consulting existing scholarship will: (a) miss existing interpretations, (b) duplicate known work, (c) fail to identify cross-references available only in secondary literature. However, accepting a published reading without K-criteria validation is equally wrong — it replaces one type of error (ignorance) with another (uncritical acceptance).

### 3.1 Step-by-Step Algorithm

When presented with a Linear A tablet, follow these steps IN ORDER:

```
STEP 0: PALAEOGRAPHIC VERIFICATION (K7) [v8.0]
  → BEFORE any interpretation, verify you are reading the CORRECT physical signs
  → Compare photograph, facsimile drawing, and GORILA transcription
  → If a sign fails K7, mark it and reduce confidence accordingly
  → If K7 cannot be performed (no image available), state this explicitly
  → Rationale: The AR1 incident proved that AI models exhibit "lexical bias" — 
    reading signs they EXPECT rather than signs that ARE THERE

STEP 1: IDENTIFY THE KNOWN ELEMENTS
  → Separate ideograms (CYP, OLE, VIN, GRA, E, VIR, etc.) from syllabograms
  → Identify numbers and fractions
  → Mark what you KNOW vs what is UNKNOWN

STEP 2: ANALYZE THE HEADER
  → First line usually contains: personal name / place name + transaction sign
  → The header tells you WHAT TYPE of tablet this is

STEP 3: CHECK FOR ARITHMETIC
  → Do the numbers add up?
  → Is there a KU-RO (total) at the end?
  → Does the number after KU-RO = sum of individual entries?
  → This is your STRONGEST independent evidence

STEP 4: IDENTIFY STRUCTURAL POSITION
  → Beginning of list = header/category
  → Middle entries = items/persons + quantities
  → End of list = total (KU-RO) or deficit (KI-RO)
  → Position determines function INDEPENDENTLY of phonetics

STEP 5: CROSS-REFERENCE WITH OTHER TABLETS
  → Search for the same sign/sequence in other tablets
  → Same sign in same position across 3+ tablets = confirmed function
  → This is MORE reliable than phonetic analysis

STEP 6: APPLY R→L RULE (only if criteria met — see Section 4)
  → Check if sequence contains R-series signs
  → Apply Calibration Protocol K1-K6
  → Only proceed if K2+K3 are satisfied

STEP 7: CHECK ALL THREE LAYERS
  → Layer 1 (Aegean): pre-Greek substrate? Greek cognate?
  → Layer 3 (Semitic): Hebrew/Akkadian/Phoenician root?
  → Layer 2 (Hamitic): structural element? determinative?
  → Best result = convergence of 2+ layers

STEP 8: ASSIGN CONFIDENCE SCORE
  → Use the scoring system in Section 10
  → Mark everything with explicit percentage
  → Explain your reasoning
```

## SECTION 4 — THE R→L RULE WITH CALIBRATION PROTOCOL

### 4.1 What the Rule Is

Linear A/B CANNOT distinguish /r/ from /l/ — they use the SAME sign series for both. The Cypriot Syllabary (the deciphered descendant) DOES distinguish them, and the sign that corresponds to Linear A/B RO is identified AS LO.

**This is not a hypothesis — it is a documented fact of script history** (Steele, CREWS Project, Cambridge: "ro (=lo in the Cypriot Syllabary but ro/lo in Linear B)").

When we find a Semitic root with /l/ written in Linear A, the script NECESSARILY writes it with R-series signs. The "rule" is an unavoidable consequence of the writing system — not an analyst's choice.

### 4.2 The Anchor: KU-RO

KU-RO → KU-LO = כֹּל (kol) = "total" — this is the GOLD STANDARD that calibrates the rule.

**Six independent criteria confirm it:**

| Criterion | Description | Status |
|-----------|------------|--------|
| **K1 — Corpus Frequency** | 30+ tablets, 5+ sites, 500 years of use | ✅ |
| **K2 — Structural Position** | ALWAYS end of accounting list, ALWAYS before final number | ✅ |
| **K3 — Semitic Root K-L** | כֹּל (kol) = "all, total" — pan-Semitic, zero ambiguity | ✅ |
| **K4 — Cypriot Rosetta** | Academic documentation: "ro (=lo in the Cypriot Syllabary)" | ✅ |
| **K5 — Arithmetic Proof** | Number after KU-RO = sum of individual items in 30+ tablets | ✅ |
| **K6 — No Alternative** | No K-R root means "total" in any related language | ✅ |

### 4.3 Application Rules

**APPLY R→L when:**

| Criteria Met | Status |
|-------------|--------|
| K2 + K3 + 2 or more of K1/K4/K5/K6 | ✅ **Confirmed** |
| K2 + K3 + 1 of K1/K4/K5/K6 | ⚠️ **Probable** |
| K3 only OR K2 only | ❓ **Working hypothesis** |
| Neither K2 nor K3 | ❌ **Rejected — do NOT apply** |

**CRITICAL:** Without K2 (structural position) OR K3 (identifiable root), R→L is NOT applied. This prevents over-application.

### 4.4 The R→L Series Extensions

### 6.2 Example B — MEDIUM: AR Zf 1 (Title Inscription)

**Tablet:** AR Zf 1, Gold pin from Archanes, ~1600 BCE  
**Raw data:** QA-KI-SE-NU-TI

**Step 1:** Five syllabograms. No ideograms, no numbers.  
**Step 2:** This is a title/dedicatory inscription (single word on prestige object).

**Step 6:** No R-series signs → R→L not needed.  
**Step 7 — Three layers:**

- Cypro-Minoan parallel: KI-SI / KE-SE in similar administrative context
- -NU-TI = Cypro-Minoan office suffix

**Conclusion:** QA-KI-SE-NU-TI = "The Enthroned One" — a title combining Semitic root with Cypro-Minoan morphology.  
**Confidence:** 85%

### 6.3 Example C — HARD: HT 35 (Pharmaceutical Ointment)

## SECTION 8 — RED LINES (What NEVER to Do)

### 8.1 Absolute Prohibitions

1. **NEVER apply R→L without K2+K3.** The calibration protocol exists specifically to prevent arbitrary application.

2. **NEVER seek a Semitic root for a divine name.** DA-MA-TE is NOT Semitic — it is Layer 1 (Aegean). Theonyms belong to the substrate.

3. **NEVER ignore the numbers.** Arithmetic is language-independent and is your STRONGEST witness. If numbers don't add up, your reading is wrong.

4. **NEVER trust a single tablet.** You need AT LEAST 3 cross-references for any confirmed reading. Hapax legomena (words appearing only once) receive automatic penalty.

5. **NEVER assume "ritual" means "religious."** Tablets previously classified as "ritual" may record pharmaceutical, medical, or industrial activities (see Section 9).

6. **NEVER claim certainty above your evidence.** Use the confidence scoring system (Section 10) honestly. A 60% reading marked as 60% is more valuable than a 60% reading presented as 90%.

7. **NEVER apply R→L to unknown signs (\*301, etc.).** If the phonetic value is unknown, the rule cannot be tested.

8. **NEVER apply the Cypriot chain to Disc-only signs.** The chain requires Linear A as an intermediate step. Signs unique to the Phaistos Disc cannot be validated through Cyprus.

9. **NEVER analyze a tablet without checking existing bibliography first.** [v8.1 — NEW] No serious scholar examines an inscription without first consulting GORILA, Younger, SigLA, and published literature. An AI that skips this step will: (a) miss existing interpretations that provide direction, (b) waste effort duplicating known work, (c) fail to identify cross-references available only in secondary literature. **However:** published interpretations enter the method as **hypotheses to test**, not as validated readings. A scholarly proposal that does not satisfy K-criteria remains at k\*<0.50 regardless of who proposed it. The bibliographic check tells you WHERE TO LOOK — the K-criteria tell you WHAT IS TRUE.

### 8.2 Common Errors to Avoid

| Error | Why It's Wrong | Correct Approach |
|-------|---------------|-----------------|
| Finding a Semitic root for every sequence | Many sequences are Layer 1 (Aegean) or unknown | Check Layer 1 FIRST |
| Applying R→L to all R-signs automatically | Some R's are genuine R's (toponyms, etc.) | Use K1-K6 protocol |
| Treating fractional quantities as errors | Fractions = pharmaceutical/measured substances | Recognize pharmaceutical context |
| Ignoring sign position | Position is independent of phonetics | Always check structural position first |
| Over-interpreting hapax legomena | Single attestation = insufficient evidence | Mark as "insufficient data" |
| Skipping bibliographic check | Misses existing scholarship, redundant analysis | Always check GORILA/Younger/SigLA/literature FIRST |

# SECTION 11 — BEHAVIORAL ARCHAEOLOGY METHOD
## "When Everything Else Fails, Let the Tablet Speak"
### Caphtor-Minoan Training Protocol — Supplementary Section
**Version 1.0 — February 2026**
**Origin:** SA-RA₂ case study, Hagia Triada corpus analysis

---

## 11.0 WHEN TO USE THIS METHOD

Apply the Behavioral Archaeology Method when:

- Steps 1–8 of the standard protocol have been applied and no reading above k*=0.50 has been achieved
- The sequence is too frequent to be a hapax but too ambiguous to be identified phonologically
- Layer 1, Layer 2, and Layer 3 analysis have all returned either silence or conflicting results
- You have access to 3+ tablets containing the same sequence

**Critical rule:** This method does NOT replace the standard protocol. It runs IN PARALLEL with it, using only structural and quantitative evidence. Its output is a **functional description**, not a phonological reading. A functional description that passes K1–K7 is more valuable than a phonological reading that does not.

---

## 11.1 THE CORE PRINCIPLE

> **"If you cannot name it, describe what it does. If the description is consistent across 3+ tablets, you have found something real."**

Standard decipherment asks: *What does this sequence sound like? What language does it come from?*

The Behavioral Archaeology Method asks instead: *What does this sequence DO? Where does it appear? What comes with it? What does it control?*

These are different questions. The second set can be answered from the tablets alone, without phonology, without etymology, without bilingual texts.

The method was developed during the SA-RA₂ analysis (Hagia Triada, February 2026), where 20+ tablets across 9 scribes, 2 buildings, and 350 years contained a sequence that resisted all standard identification — until the tablets themselves revealed its function through systematic behavioral observation.

---

## 11.2 THE FIVE BEHAVIORAL QUESTIONS

When you encounter an unidentified sequence appearing 3+ times, ask these five questions in order. Do not skip ahead. Do not begin phonological analysis until all five are answered.

---

### QUESTION 1 — WHAT IS ITS SYNTACTIC POSITION?

Examine every occurrence and map where the sequence sits relative to ideograms and numbers.

```
PATTERN A:  [SEQUENCE] · [IDEOGRAM] [NUMBER]
PATTERN B:  [IDEOGRAM] [SEQUENCE] [NUMBER]
PATTERN C:  [IDEOGRAM] [NUMBER] [SEQUENCE]
PATTERN D:  [SEQUENCE] alone, with = sign
```

**What position tells you:**
- If always BEFORE the ideogram (Pattern A): the sequence OWNS or GOVERNS the commodity
- If always AFTER the ideogram (Pattern B): the sequence DESCRIBES the commodity (qualifier)
- If always AFTER the number (Pattern C): the sequence is a closing term (total, deficit)
- If alone with = (Pattern D): the sequence is a label or category header

**Red flag:** If the sequence changes position across tablets, it may be two different tokens with similar transcription. Check K7 immediately.

**SA-RA₂ result:** Pattern A consistently — it always precedes the ideogram. This alone eliminates commodity qualifier, closing term, and label interpretations.

---

### QUESTION 2 — HOW MANY DIFFERENT COMMODITIES DOES IT APPEAR WITH?

List every ideogram that follows the sequence across all tablets.

```
1 commodity only   → probable commodity qualifier or type marker
2–3 commodities    → possible title or personal name
4+ commodities     → CANNOT be a commodity qualifier
                    → strong candidate for institutional entity
```

**Critical logic:** A commodity qualifier (like OLE+RI or OLE+TA) describes ONE type of commodity. An entity that appears with grain, oil, aromatics, and human labor simultaneously is not a qualifier — it is something that HAS or CONTROLS those things.

**SA-RA₂ result:** 6+ different ideograms (GRA, CYP, OLE+DI, VIR+[?], *308, *401+*304). This definitively eliminates commodity qualifier.

---

### QUESTION 3 — WHAT IS ITS RELATIONSHIP WITH KU-RO AND KI-RO?

Three possible relationships:

```
RELATIONSHIP A — Appears IN the list, before KU-RO
  → The sequence is one of the items being totaled
  → Function: recipient, contributor, or category within the accounting

RELATIONSHIP B — Appears AFTER KU-RO
  → The sequence is OUTSIDE the main accounting
  → Function: separate allocation, set-aside, or privileged category
  → Parallel: Mesopotamian sattukku (temple portion removed before distribution)

RELATIONSHIP C — KI-RO appears for the SAME COMMODITY as the sequence
  → The sequence has obligations that were not met
  → Function: an entity with contractual duties, not merely a recipient
```

**What this tells you:** Only an institutional entity appears in all three relationships across different tablets. A personal name appears once in one relationship. A commodity qualifier has no relationship with KU-RO/KI-RO at all.

**SA-RA₂ result:** All three relationships documented. Post-KU-RO position (HT100) + KI-RO deficit against it (HT30) + standard list position (HT102) = institutional entity with obligations.

---

### QUESTION 4 — WHAT IS THE QUALITY PROFILE OF ITS COMMODITIES?

For each commodity the sequence receives, assess:

```
MASS COMMODITIES (GRA, VIN in large quantities):
  → Indicates maintenance of dependent population
  → Scale = number of people being fed

SPECIALIZED COMMODITIES (CYP, *308, aromatic compounds):
  → Small quantities, high value, pharmaceutical/ritual use
  → Indicates production or consumption of specialized goods

RITUAL/SACRED COMMODITIES (OLE+DI with DA-TE association):
  → Indicates religious function or priestly role

HUMAN RESOURCES (VIR+[?], MUL+[?]):
  → Indicates administrative authority over people
  → This is the strongest indicator of institutional status
```

**The decisive combination:** An entity that simultaneously receives mass commodities (to feed people), specialized aromatics (to produce or use), sacred oils (ritual function), AND human labor (administrative authority) can only be an institution — not a person, not a place name, not a commodity type.

**SA-RA₂ result:** All four categories present. This combination is uniquely consistent with a temple-palatial complex operating as an autonomous economic unit.

---

### QUESTION 5 — DOES IT APPEAR MORE THAN ONCE IN THE SAME TABLET?

```
0–1 occurrences per tablet  → consistent with personal name or place
2+ occurrences same tablet  → INCONSISTENT with personal name
                            → consistent with: accounting category,
                              institutional code, transaction type
```

**Why this matters:** A personal name appears once per accounting record. A place name appears once. But a ledger category (like "Temple account" or "Palace allocation") can appear multiple times in the same document, once for each commodity type being allocated to it.

**SA-RA₂ result:** HT114a has SA-RA₂ GRA 10 AND SA-RA₂ GRA 5 as separate entries. This is the strongest single-tablet argument against personal name interpretation.

## 11.3 THE SYNTHESIS MATRIX

After answering all five questions, fill in this matrix:

```
BEHAVIORAL MATRIX FOR [SEQUENCE]
═══════════════════════════════════════════════════════

═══════════════════════════════════════════════════════

MATRIX INTERPRETATION:
─────────────────────
Pattern A + 1 commodity + In-list + Mass only + Single
  → COMMODITY QUALIFIER (e.g. OLE+RI, OLE+TA)

Pattern A + 1–2 commodities + In-list + Mass + Single
  → PERSONAL NAME or TITLE OF INDIVIDUAL

Pattern A + 3+ commodities + In-list + Mass + Single
  → PLACE NAME or WORKSHOP

Pattern A + 4+ commodities + Post-KU-RO + All profiles + Multi
  → INSTITUTIONAL ENTITY (temple, palace section, temenos)

Pattern C + 1 commodity + End-of-list + N/A + Single
  → ACCOUNTING TERM (KU-RO type)
```

## 11.4 FROM FUNCTION TO HYPOTHESIS

Once the behavioral matrix is complete and an institutional type is identified, THEN — and only then — return to the standard protocol:

```
STEP A: Name the function in neutral terms
  (e.g., "autonomous economic unit with ritual and administrative role")

STEP B: Search Layer 1 for Aegean institutional vocabulary
  (e.g., pre-Greek *temenos, *wanax-related terms)

STEP C: Search Layer 3 for Semitic administrative vocabulary
  (e.g., šarrum, sattukku, bīt-ilim)

STEP D: Check Cypriot chain for structural parallels
  (not phonological — structural: does this function appear in CM records?)

STEP E: Check Linear B for Mycenaean continuity
  (does this institution have a named equivalent 250 years later?)

STEP F: Assign k* based on K1–K7, NOT on behavioral analysis alone
```

**Critical warning:** The behavioral analysis generates a FUNCTIONAL HYPOTHESIS. It does NOT generate K-criteria scores by itself. The k* score must be earned through the standard protocol. A behavioral description at k*=0.00 (before K1–K7 testing) is still more honest than a phonological reading at k*=0.60 presented as k*=0.85.

## 11.5 WORKED EXAMPLE — SA-RA₂

This section documents the complete application of the Behavioral Archaeology Method to SA-RA₂, as a reference case.

### Input data

20+ tablets, Hagia Triada and Khania, ~1500–1450 BCE. Standard protocol returned no phonological reading above k*=0.50. The sequence appeared with too many different ideograms to be a commodity qualifier and persisted across too many scribes and centuries to be a personal name.

### Five questions applied

**Q1 — Syntactic position:** Pattern A exclusively. SA-RA₂ always precedes the ideogram. Never follows it.

**Q2 — Commodity diversity:** GRA, CYP, OLE+DI, VIR+[?], *308, *401+*304 — six or more distinct categories. Rules out commodity qualifier definitively.

**Q3 — KU-RO/KI-RO relationship:**
- HT100: SA-RA₂ CYP 5¼ appears AFTER KU-RO 97 → post-total set-aside
- HT30: KI-RO·CYP 8 follows a list beginning SA-RA₂ CYP 14½ → deficit obligation
- HT102: SA-RA₂ GRA 976 is first entry in list before KU-RO 1060 → standard allocation

All three relationship types present. This is unique in the corpus.

**Q4 — Quality profile:**
- Mass commodities: GRA 976, GRA 41, GRA 20 → feeds a large population
- Specialized aromatics: CYP 14½, CYP 4, *308 ½ → pharmaceutical/ritual production
- Sacred oil: OLE+DI 1 → associated elsewhere with DA-TE (divine context)
- Human labor: VIR+[?] 235 → controls a workforce comparable to KA-PA (VIR+[?] 234)

**Q5 — Multi-occurrence:** HT114a: SA-RA₂ GRA 10 AND SA-RA₂ GRA 5 in same tablet.

### Behavioral matrix result

```
Q1: Pattern A (governs commodity)
Q2: 6+ commodities (cannot be qualifier)
Q3: All three relationships (A + B + C)
Q4: All four profiles (Mass + Specialized + Ritual + Human)
Q5: Yes (HT114a)
```

### Functional description (before K-criteria testing)

> SA-RA₂ is the ledger code for an autonomous institutional entity that: maintains a dependent population (GRA at scale), produces or consumes specialized aromatic and pharmaceutical goods (CYP, *308), holds ritual/sacred oil allocations (OLE+DI), commands a substantial workforce (VIR+[?] 235), holds set-aside allocations outside the main accounting (post-KU-RO position), and carries contractual obligations recorded as deficits when unmet (KI-RO).

This functional description matches one known type of Bronze Age institution: the **temple-palatial complex** operating as an independent economic unit — the Minoan predecessor of what Linear B calls **te-me-no** (temenos).

### K-criteria testing result

After applying the functional hypothesis to the standard protocol:

```
K1 (corpus frequency): ✅ 20+ tablets, 9 scribes, 2 cities, 350 years
K2 (structural position): ✅ Pattern A consistently
K3 (Semitic root šar/šarrum): ✅ pan-Semitic administrative title
K4 (Cypriot chain): ⚠️ SA-RA-NI in Enkomi cylinder — structural parallel,
                       phonological value not confirmed (-7%)
K5 (arithmetic): ⚠️ HT102 wrong total — cannot fully verify
K6 (no alternative): ✅ no competing reading explains all five behavioral patterns
K7 (palaeographic): ✅ consistent across all 9 scribes

Final k* = 0.82 (Probable/Strong)
```

### Residual uncertainty

The subscript ₂ (RA₂ vs RA) remains unexplained. This may indicate a palatalized form /rya/ or /la/, which could affect the Semitic etymology. The behavioral description is not affected by this uncertainty — the function is clear regardless of exact phonology. Confidence in the FUNCTIONAL reading: 0.88. Confidence in the PHONOLOGICAL reading: 0.72.

## 11.6 THE PRINCIPLE OF BEHAVIORAL PRIORITY

The SA-RA₂ case established a general principle for Linear A analysis:

> **When a sequence is too frequent, too widespread, and too variable to fit any standard category, its behavior across the corpus IS its identity. The function precedes the name.**

This is not a weakness of the method — it is an acknowledgment that administrative systems in any language create functional categories that are more stable than the words used to name them. The Minoan scribes may have changed dynasties, languages, and even palaces — but the ledger category for "temple allocation" persisted for centuries because the institution persisted.

Finding the function is finding the thing. The name is secondary.
