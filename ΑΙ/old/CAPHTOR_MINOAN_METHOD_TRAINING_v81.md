## SECTION 1 — THE TRI-LAYERED MODEL

### 1.1 Core Thesis

Minoan Crete was NOT linguistically homogeneous. Linear A records three distinct sociolinguistic strata:

| Layer | Population | Biblical Source | Date | Contribution |
|-------|-----------|----------------|------|-------------|
| **Layer 1 — Substrate** | Japhethites / Pelasgians | Gen 10: Japheth → Javan | ~3000-2000 BCE | Vowels, divine names, acrophony, plant terminology |
| **Layer 2 — Superstate** | Hamites / Caphtorim | Gen 10: Ham → Mizraim → Caphtorim | ~2000-1700 BCE | Writing structure, CV-syllables, determinatives, bureaucracy |
| **Layer 3 — Influence** | Semitic traders | Phoenicians, Canaanites | ~1800-1400 BCE | Commercial vocabulary: KU-RO, M-L-K, Š-L-M |

### 1.2 The Substrate Law

Universal linguistic law: **religious vocabulary** (oldest layer) survives longest, while **commercial vocabulary** is replaced first. This explains why Linear A has:
- Aegean divine names (DA-MA-TE) — Layer 1, survived into Greek
- Semitic commercial roots (KU-RO) — Layer 3, replaced by Greek after ~1450 BCE

### 1.3 How to Apply the Model

When analyzing a Linear A sequence:

1. **First check Layer 1 (Aegean/Greek):** Does the sequence match a known pre-Greek substrate word or a Greek word with pre-Greek etymology? Look for -nthos, -ssos, -aro endings.
2. **Then check Layer 3 (Semitic):** Does the sequence, after applying R→L where valid, match a Semitic root (Hebrew, Akkadian, Phoenician)? Especially for commercial/administrative terms.
3. **Consider Layer 2 (Hamitic/Egyptian):** Is the sign a determinative or structural element borrowed from Egyptian writing practice?
4. **Best result:** When all three layers converge on the same reading.

### 1.4 What NOT to Assume

- Linear A is NOT a Semitic language — it has Semitic LOANWORDS
- Linear A is NOT Greek — it has pre-Greek SUBSTRATE words
- The writing SYSTEM was designed by Caphtorim (Layer 2) for an Aegean language (Layer 1) enriched with Semitic trade terms (Layer 3)

## SECTION 3 — TABLET READING PROTOCOL

### 3.0 Research Prerequisites [v8.1 — NEW]

**This section describes what to do BEFORE applying the decipherment algorithm. These are not analytical steps of the method — they are standard academic practice. They do not generate k\* scores or satisfy K-criteria. They provide CONTEXT, not CONCLUSIONS.**

Before applying Steps 0-8 to any tablet, consult existing scholarship:

| Source | What it provides | URL / Reference |
|--------|-----------------|-----------------|
| GORILA (Godart & Olivier) | Official transcription, facsimile, notes | Vols. 1-5 (print) |
| Younger's commentary | Phonetic transcription, scholarly notes per tablet | web archive / lineara.xyz |
| SigLA (Salgarella) | Palaeographic sign data, searchable | sigla.phis.me |
| Academic literature | Published interpretations, parallels | Scholar search / Academia.edu |

**Rules:**
- If prior interpretations exist, note them — but they enter the method as **hypotheses to test**, not as validated readings
- A published interpretation (e.g. Mosenkis 2015: KU-PI = "Cypriot") provides a **direction to investigate**, not a K-criteria score
- Only the K1-K7 criteria can validate a reading — scholarly consensus without K-criteria remains at the hypothesis level
- If you have web search capability, use it — but clearly separate what you FOUND from what the METHOD VALIDATES

**Rationale:** The ZA14/KU-PI incident (Feb 2026) demonstrated that an AI model following the Protocol mechanically but without consulting existing scholarship will: (a) miss existing interpretations, (b) duplicate known work, (c) fail to identify cross-references available only in secondary literature. However, accepting a published reading without K-criteria validation is equally wrong — it replaces one type of error (ignorance) with another (uncritical acceptance).

### 3.1 Step-by-Step Algorithm

When presented with a Linear A tablet, follow these steps IN ORDER:

```
STEP 0: PALAEOGRAPHIC VERIFICATION (K7) [v8.0]
  → BEFORE any interpretation, verify you are reading the CORRECT physical signs
  → Compare photograph, facsimile drawing, and GORILA transcription
  → If a sign fails K7, mark it and reduce confidence accordingly
  → If K7 cannot be performed (no image available), state this explicitly
  → Rationale: The AR1 incident proved that AI models exhibit "lexical bias" — 
    reading signs they EXPECT rather than signs that ARE THERE

STEP 1: IDENTIFY THE KNOWN ELEMENTS
  → Separate ideograms (CYP, OLE, VIN, GRA, E, VIR, etc.) from syllabograms
  → Identify numbers and fractions
  → Mark what you KNOW vs what is UNKNOWN

STEP 2: ANALYZE THE HEADER
  → First line usually contains: personal name / place name + transaction sign
  → The header tells you WHAT TYPE of tablet this is

STEP 3: CHECK FOR ARITHMETIC
  → Do the numbers add up?
  → Is there a KU-RO (total) at the end?
  → Does the number after KU-RO = sum of individual entries?
  → This is your STRONGEST independent evidence

STEP 4: IDENTIFY STRUCTURAL POSITION
  → Beginning of list = header/category
  → Middle entries = items/persons + quantities
  → End of list = total (KU-RO) or deficit (KI-RO)
  → Position determines function INDEPENDENTLY of phonetics

STEP 5: CROSS-REFERENCE WITH OTHER TABLETS
  → Search for the same sign/sequence in other tablets
  → Same sign in same position across 3+ tablets = confirmed function
  → This is MORE reliable than phonetic analysis

STEP 6: APPLY R→L RULE (only if criteria met — see Section 4)
  → Check if sequence contains R-series signs
  → Apply Calibration Protocol K1-K6
  → Only proceed if K2+K3 are satisfied

STEP 7: CHECK ALL THREE LAYERS
  → Layer 1 (Aegean): pre-Greek substrate? Greek cognate?
  → Layer 3 (Semitic): Hebrew/Akkadian/Phoenician root?
  → Layer 2 (Hamitic): structural element? determinative?
  → Best result = convergence of 2+ layers

STEP 8: ASSIGN CONFIDENCE SCORE
  → Use the scoring system in Section 10
  → Mark everything with explicit percentage
  → Explain your reasoning
```

### 3.2 What Numbers Tell You (Independent of Language)

| VIR + number / GRA + number | Workers + ration | Personnel records |

## SECTION 4 — THE R→L RULE WITH CALIBRATION PROTOCOL

### 4.1 What the Rule Is

Linear A/B CANNOT distinguish /r/ from /l/ — they use the SAME sign series for both. The Cypriot Syllabary (the deciphered descendant) DOES distinguish them, and the sign that corresponds to Linear A/B RO is identified AS LO.

**This is not a hypothesis — it is a documented fact of script history** (Steele, CREWS Project, Cambridge: "ro (=lo in the Cypriot Syllabary but ro/lo in Linear B)").

When we find a Semitic root with /l/ written in Linear A, the script NECESSARILY writes it with R-series signs. The "rule" is an unavoidable consequence of the writing system — not an analyst's choice.

### 4.2 The Anchor: KU-RO

KU-RO → KU-LO = כֹּל (kol) = "total" — this is the GOLD STANDARD that calibrates the rule.

**Six independent criteria confirm it:**

| Criterion | Description | Status |
|-----------|------------|--------|
| **K1 — Corpus Frequency** | 30+ tablets, 5+ sites, 500 years of use | ✅ |
| **K2 — Structural Position** | ALWAYS end of accounting list, ALWAYS before final number | ✅ |
| **K3 — Semitic Root K-L** | כֹּל (kol) = "all, total" — pan-Semitic, zero ambiguity | ✅ |
| **K4 — Cypriot Rosetta** | Academic documentation: "ro (=lo in the Cypriot Syllabary)" | ✅ |
| **K5 — Arithmetic Proof** | Number after KU-RO = sum of individual items in 30+ tablets | ✅ |
| **K6 — No Alternative** | No K-R root means "total" in any related language | ✅ |

### 4.3 Application Rules

**APPLY R→L when:**

| Criteria Met | Status |
|-------------|--------|
| K2 + K3 + 2 or more of K1/K4/K5/K6 | ✅ **Confirmed** |
| K2 + K3 + 1 of K1/K4/K5/K6 | ⚠️ **Probable** |
| K3 only OR K2 only | ❓ **Working hypothesis** |
| Neither K2 nor K3 | ❌ **Rejected — do NOT apply** |

**CRITICAL:** Without K2 (structural position) OR K3 (identifiable root), R→L is NOT applied. This prevents over-application.

### 4.4 The R→L Series Extensions

The rule applies to the ENTIRE liquid series:

| Linear A/B Sign | R-reading | L-reading (via Cypriot) | Examples |
|----------------|-----------|----------------------|---------|
| RO series | ro | **lo** | KU-RO → KU-LO |
| RA series | ra | **la** | A-SA-SA-RA-ME → A-SA-SA-LA-ME |
| RE series | re | **le** | A-RE-PI → A-LE-PI; RE-NA → LE-NA |
| RI series | ri | **li** | I-RI-DE-KA → I-LI-DE-KA |
| RU series | ru | **lu** | SI-RU-TE → SI-LU-TE |

### 4.5 When R→L is REJECTED

| Case | Reason | Example |
|------|--------|---------|
| Toponyms with known R | Cypriot evidence preserves R | TU-RU-SA (Tarsos) |
| Unknown signs (\*301 etc.) | Cannot assess without phonetic value | TA-NA-I-\*301-TI |

## SECTION 6 — WORKED EXAMPLES

### 6.1 Example A — EASY: HT 86 (Agricultural Accounting)

**Tablet:** HT 86, Hagia Triada, Late Minoan IB  
**Raw data:** GRA 10 | OLE 5 | FIC 20 | KU-RO [Total: 35]

**Step 1 — Known elements:** GRA (grain) = 10, OLE (oil) = 5, FIC (figs) = 20, KU-RO = ?  
**Step 3 — Arithmetic:** 10 + 5 + 20 = 35. Number after KU-RO = 35. ✅ Match.  
**Step 4 — Position:** KU-RO is at the END of the list, before the final number.  
**Conclusion:** KU-RO = "Total." Arithmetic proof is independent of language assumption.  
**Confidence:** 95%

### 6.2 Example B — MEDIUM: AR Zf 1 (Title Inscription)

**Tablet:** AR Zf 1, Gold pin from Archanes, ~1600 BCE  
**Raw data:** QA-KI-SE-NU-TI

**Step 1:** Five syllabograms. No ideograms, no numbers.  
**Step 2:** This is a title/dedicatory inscription (single word on prestige object).  
**Step 5:** Cross-reference: QA- prefix appears in Cypro-Minoan; -TI suffix appears in office titles.  
**Step 6:** No R-series signs → R→L not needed.  
**Step 7 — Three layers:**
- Layer 3 (Semitic): QA- = Semitic relative pronoun "he who..." + KI-SE = כִּסֵּא (kisse) = THRONE
- Cypro-Minoan parallel: KI-SI / KE-SE in similar administrative context
- -NU-TI = Cypro-Minoan office suffix

**Conclusion:** QA-KI-SE-NU-TI = "The Enthroned One" — a title combining Semitic root with Cypro-Minoan morphology.  
**Confidence:** 85%

### 6.3 Example C — HARD: HT 35 (Pharmaceutical Ointment)

**Tablet:** HT 35, Villa Magazine, Hagia Triada  
**Raw data:** TI-TI-KU · \*326 | I-KU-TA CYP 1 | MI+JA+RU ⅓ | OLE+RI ⅓ | \*308 ⅓ | OLE+TA | E 5 | QA₂+[?]+PU ¹⁄₁₆ | VIN ⅕

**Step 1 — Known elements:**
- CYP = cyperus (known ideogram) = 1 unit
- OLE+RI, OLE+TA = two types of olive oil
- \*308 = coriander (Schoep 2002) = ⅓
- E = figs = 5
- VIN = wine/vinegar = ⅕
- MI+JA+RU = compound ligature = ⅓
- QA₂+[?]+PU = compound ligature = ¹⁄₁₆

**Step 2 — Header:** TI-TI-KU · \*326 = title/category of recipe. I-KU-TA = dative "for the KU-TA [workers]"

**Step 3 — Arithmetic:** All quantities are FRACTIONS — not typical for food rations (which use integers). This suggests pharmaceutical dosages.

**Step 5 — Cross-reference:**
- CYP + \*308 + MI+JA+RU appear TOGETHER in HT23a, HT35, KH12 (3 tablets, 2 cities)
- OLE+RI always in 1:1 ratio with MI+JA+RU (HT35: ⅓=⅓; HT60: ¹⁄₁₆=¹⁄₁₆; HT23a: ≈⅙=≈⅙)
- QA₂+[?]+PU traded by WEIGHT in HT45b (double mina) → solid substance (resin)

**Step 7 — Three layers:**
- Layer 1 (Aegean): CYP = κύπειρος (pre-Greek root \*kup-); "akisaro" (Cretan dialect for Cistus/ladanum) = pre-Greek substrate
- Layer 3 (Semitic): MI+JA+RU ← מֹר (mor) = myrrh; KU-RO/KI-RO accounting terms
- Layer 2 (Hamitic): ideographic structure, fractional system, four-tier hierarchy

**Step 8 — Conclusion:** HT35 = pharmaceutical ointment recipe (aleiphar) for workers, NOT a ritual offering. This is the Minoan predecessor of the Mycenaean perfume industry documented by Shelmerdine (1985) at Pylos.

**Confidence:** 85% for pharmaceutical interpretation; individual ingredients 70-95%.

### 6.4 Example D — REJECTED: TU-RU-SA

**Tablet:** KO Za 1  
**Raw data:** TU-RU-SA

**Step 6 — R→L check:**
- RU present → test R→L

## SECTION 8 — RED LINES (What NEVER to Do)

### 8.1 Absolute Prohibitions

1. **NEVER apply R→L without K2+K3.** The calibration protocol exists specifically to prevent arbitrary application.

2. **NEVER seek a Semitic root for a divine name.** DA-MA-TE is NOT Semitic — it is Layer 1 (Aegean). Theonyms belong to the substrate.

3. **NEVER ignore the numbers.** Arithmetic is language-independent and is your STRONGEST witness. If numbers don't add up, your reading is wrong.

4. **NEVER trust a single tablet.** You need AT LEAST 3 cross-references for any confirmed reading. Hapax legomena (words appearing only once) receive automatic penalty.

5. **NEVER assume "ritual" means "religious."** Tablets previously classified as "ritual" may record pharmaceutical, medical, or industrial activities (see Section 9).

6. **NEVER claim certainty above your evidence.** Use the confidence scoring system (Section 10) honestly. A 60% reading marked as 60% is more valuable than a 60% reading presented as 90%.

7. **NEVER apply R→L to unknown signs (\*301, etc.).** If the phonetic value is unknown, the rule cannot be tested.

8. **NEVER apply the Cypriot chain to Disc-only signs.** The chain requires Linear A as an intermediate step. Signs unique to the Phaistos Disc cannot be validated through Cyprus.

9. **NEVER analyze a tablet without checking existing bibliography first.** [v8.1 — NEW] No serious scholar examines an inscription without first consulting GORILA, Younger, SigLA, and published literature. An AI that skips this step will: (a) miss existing interpretations that provide direction, (b) waste effort duplicating known work, (c) fail to identify cross-references available only in secondary literature. **However:** published interpretations enter the method as **hypotheses to test**, not as validated readings. A scholarly proposal that does not satisfy K-criteria remains at k\*<0.50 regardless of who proposed it. The bibliographic check tells you WHERE TO LOOK — the K-criteria tell you WHAT IS TRUE.

### 8.2 Common Errors to Avoid

| Error | Why It's Wrong | Correct Approach |
|-------|---------------|-----------------|
| Finding a Semitic root for every sequence | Many sequences are Layer 1 (Aegean) or unknown | Check Layer 1 FIRST |
| Applying R→L to all R-signs automatically | Some R's are genuine R's (toponyms, etc.) | Use K1-K6 protocol |
| Treating fractional quantities as errors | Fractions = pharmaceutical/measured substances | Recognize pharmaceutical context |
| Ignoring sign position | Position is independent of phonetics | Always check structural position first |
| Over-interpreting hapax legomena | Single attestation = insufficient evidence | Mark as "insufficient data" |
| Skipping bibliographic check | Misses existing scholarship, redundant analysis | Always check GORILA/Younger/SigLA/literature FIRST |

## SECTION 10 — CONFIDENCE SCORING SYSTEM

### 10.1 Score Categories

| Score | Label | Meaning |
|-------|-------|---------|
| 95-100% | ✅ **Proven** | Mathematical proof or confirmed by Linear B |
| 85-94% | ✅ **Strong** | Multiple independent confirmations |
| 70-84% | ⚠️ **Probable** | Good evidence, some uncertainty |
| 50-69% | ❓ **Working hypothesis** | Plausible but needs more evidence |
| Below 50% | ❌ **Insufficient** | Do not present as finding |

### 10.2 Factors That INCREASE Confidence

| Factor | Bonus | Example |
|--------|-------|---------|
| Arithmetic verification | +20% | KU-RO sum matches |
| Cross-reference 3+ tablets | +15% | CYP+\*308+MI+JA+RU in 3 tablets |
| Linear B confirmation | +20% | A-RE-PI = a-re-pa |
| Cypriot chain positive | +15% | R→L confirmed |
| Two layers converge | +10% | Semitic root + structural position |
| Three layers converge | +20% | All three layers agree |

### 10.3 Factors That DECREASE Confidence

| Factor | Penalty | Example |
|--------|---------|---------|
| Hapax legomenon (1 attestation) | -15% | TU-RU-SA |
| Cypriot silence | -10% | QA-RA2-WA |
| Cypriot negative | -30% | Toponym with known R |
| Unknown sign (\*301 etc.) | -20% | TA-NA-I-\*301-TI |
| Only one layer | -10% | Semitic root only, no structural confirmation |
| Disc-only sign (Tier B) | -15% | No Linear A correspondence |

### 10.4 Reporting Template

When reporting an analysis, use this format:

```
SEQUENCE: [the Linear A sequence]

LAYER 1 (Aegean): [analysis]

```
